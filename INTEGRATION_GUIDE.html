üìñ Umumiy Ma'lumot

SuperDoc - bu zamonaviy Word hujjatlarni browser ichida tahrirlash uchun mo'ljallangan yechim. Ushbu tizim postMessage API orqali parent window bilan aloqa o'rnatadi va dinamik ravishda hujjatlarni yuklash va saqlash imkoniyatini beradi.

üéØ Asosiy Maqsad: Backend server'dan Word hujjatni yuklash, tahrirlash va o'zgarishlarni qayta saqlash jarayonini to'liq avtomatlashtirish.

üåü Asosiy Xususiyatlar

  • ‚úÖ PostMessage orqali dinamik URL konfiguratsiyasi
  • ‚úÖ JWT token autentifikatsiyasi
  • ‚úÖ Real-time hujjat yuklash va saqlash
  • ‚úÖ Loading spinner bilan foydalanuvchi tajribasi
  • ‚úÖ Xatoliklarni boshqarish va xabar berish
  • ‚úÖ Crossorigin window kommunikatsiyasi

Ma'lumot Oqimi

Parent Window Ochiladi

Foydalanuvchi "Word Hujjatni Ochish" tugmasini bosadi

Child Window Ready Signal

SuperDoc window yuklangach { status: 'ready' } yuboradi

Ma'lumot Yuborish

Parent window hujjat URL va token yuboradi

Hujjat Yuklash

SuperDoc backend'dan hujjatni fetch qiladi

Tahrirlash

Foydalanuvchi hujjatni tahrirlay oladi

Saqlash

O'zgarishlar PUT request orqali backend'ga saqlanadi

üîå API Reference

PostMessage Data Format

Parent window quyidagi formatda ma'lumot yuborishi kerak:

{
    "input_url": "http://localhost:8000/api/files/2/content/",
    "output_url": "http://localhost:8000/api/files/2/content/",
    "v3_ganiwer": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

Parametrlar

Parameter Tip Tavsif Majburiy
input_url string Hujjatni yuklash uchun backend URL ‚úÖ Ha
output_url string Hujjatni saqlash uchun backend URL ‚úÖ Ha
v3_ganiwer string JWT access token (autentifikatsiya) ‚úÖ Ha
‚ö†Ô∏è Diqqat: Token mutlaqo zarur! Backend API himoyalangan va har bir request uchun Authorization: Bearer {token} header talab qilinadi.

Status Messages

Status Yo'nalish Ma'nosi
{ status: 'ready' } Child ‚Üí Parent SuperDoc tayyor, ma'lumot yuborish mumkin
{ status: 'loaded', url: '...' } Child ‚Üí Parent Hujjat muvaffaqiyatli yuklandi

1Ô∏è‚É£ Parent Window Sozlash

var newWindow = null;

// PostMessage listener (faqat bir marta qo'shiladi)
window.addEventListener('message', function (event) {
    console.log('Parent received message:', event.data);

    // Child window tayyor bo'lganda
    if (event.data && event.data.status === 'ready' && newWindow) {
        console.log('‚úÖ Child window tayyor!');
        
        const token = getAccessToken(); // Tokenni olish
        
        var data = {
            input_url: 'http://localhost:8000/api/files/2/content/',
            output_url: 'http://localhost:8000/api/files/2/content/',
            v3_ganiwer: token,
        };

        newWindow.postMessage(data, '*');
    }

    // Hujjat yuklanganda
    if (event.data && event.data.status === 'loaded') {
        console.log('‚úÖ Hujjat yuklandi:', event.data.url);
    }
});

// Yangi oyna ochish
function openEditor() {
    const url = 'http://localhost:9094/';
    const features = 'width=1200,height=800';
    
    newWindow = window.open(url, 'SuperDocWindow', features);
}
‚ùå Xato: Listener'ni openEditor() funksiyasi ichida qo'shmang! Bu har safar yangi listener yaratadi va duplicate message'lar keladi.

Window.open() Features Parametri

Uchinchi parameter - windowFeatures - yangi oynaning ko'rinishini sozlaydi:

const features = 'width=1200,height=800,menubar=no,toolbar=no,location=no,status=no';
newWindow = window.open(url, 'SuperDocWindow', features);
Feature Tavsif Qiymatlar
width Oyna kengligi (pikselda) Raqam, masalan: 1200
height Oyna balandligi (pikselda) Raqam, masalan: 800
menubar Browser menyu barini ko'rsatish yes / no
toolbar Browser toolbar'ni (back, forward, ...) ko'rsatish yes / no
location URL address bar'ni ko'rsatish yes / no
status Pastki status bar'ni ko'rsatish yes / no
üí° Tavsiya: Document tahrirlash uchun menubar=no,toolbar=no,location=no,status=no qo'yish tavsiya etiladi - bu foydalanuvchini chalg'itmaydi va oyna toza ko'rinadi.

2Ô∏è‚É£ Backend API Endpoints

Hujjatni Yuklash (GET)

GET /api/files/{id}/content/
Authorization: Bearer {token}

Response: binary/octet-stream (Word fayl)

Hujjatni Saqlash (PUT)

PUT /api/files/{id}/content/
Authorization: Bearer {token}
Content-Type: multipart/form-data

Body: FormData with 'file' field

Response Codes:
  200-299: Muvaffaqiyat
  401: Token eskirgan / noto'g'ri
  403: Ruxsat yo'q
  601: Hujjat imzolangan (o'zgartirib bo'lmaydi)

3Ô∏è‚É£ Token Boshqaruvi

function getAccessToken() {
    // localStorage'dan olish
    return localStorage.getItem('access_token');
}

function setTokens(access, refresh) {
    localStorage.setItem('access_token', access);
    localStorage.setItem('refresh_token', refresh);
}

// Login qilish
async function login(username, password) {
    const response = await fetch('http://localhost:8000/api/token/', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ username, password })
    });
    
    const data = await response.json();
    setTokens(data.access, data.refresh);
}
